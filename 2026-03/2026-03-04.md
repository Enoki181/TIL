## Tech Log

### wgetは冤罪 → apt-keyが真犯人
#### 起きたこと
RUNTEQのヘルプ記載の`Dockerfile.dev`を参考にして環境構築をしていたところ、Docker build 時に以下のエラーが発生。
```
failed to solve: process "... wget ... | apt-key add -" did not complete successfully: exit code: 127
```
最初は `wget` が原因だと思ったが、実際は違った。

#### 事象の整理
対象Dockerfile（一部）:
```dockerfile
RUN apt-get update -qq \
&& apt-get install -y ca-certificates curl gnupg \
&& mkdir -p /etc/apt/keyrings \
&& curl -fsSL https://deb.nodesource.com/gpgkey/nodesource-repo.gpg.key | gpg --dearmor -o /etc/apt/keyrings/nodesource.gpg \
&& NODE_MAJOR=20 \
&& echo "deb [signed-by=/etc/apt/keyrings/nodesource.gpg] https://deb.nodesource.com/node_$NODE_MAJOR.x nodistro main" | tee /etc/apt/sources.list.d/nodesource.list \
&& wget --quiet -O - https://dl.yarnpkg.com/debian/pubkey.gpg | apt-key add - \
&& echo "deb https://dl.yarnpkg.com/debian/ stable main" | tee /etc/apt/sources.list.d/yarn.list
```
#### 切り分け手順

##### 1. wgetが原因か確認
```bash
docker run --rm ruby:3.4.8 bash -c "which wget"
```
→ `/usr/bin/wget` と表示される
→ wget は存在する

同様に以下も確認：
```bash
docker run --rm ruby:3.3.6 bash -c "which wget"
docker run --rm ruby:4.0.1 bash -c "which wget"
```
→ すべて存在

**👉 wgetは冤罪**

##### 2. exit code 127 の意味を確認
127 = execve failed（PATH上に実行可能ファイルが見つからない）
該当行はパイプを含む：
```
wget ... | apt-key add -
```
つまり、
- wget
- apt-key (← これを失念していた)
どちらかが存在しない可能性がある。

##### 3. apt-key を確認

```bash
docker run --rm ruby:3.4.8 bash -c "which apt-key"
```
→ 何も表示されない（存在しない）

```bash
docker run --rm ruby:3.3.6 bash -c "which apt-key"
```
→ `/usr/bin/apt-key`
**👉 ruby:3.4系では apt-key が削除されていた**

#### 結論
`exit code: 127` の原因は **apt-key が存在しなかったこと**。

ruby:3.3系では存在していたが、
ruby:3.4系ではベースイメージのDebian/Ubuntu更新に伴う仕様変更の可能性あり（未確認）

#### そもそも apt-key とは？
APT がリポジトリの GPG 署名を検証するための公開鍵を登録する仕組み

しかし、
* 登録された鍵がシステム全体で有効になる
* リポジトリ単位で制限できない
という設計上の問題があり、現在は非推奨

#### 現在の推奨方式

`/etc/apt/keyrings/` に鍵を保存し、
```
deb [signed-by=/etc/apt/keyrings/xxx.gpg] ...
```
のように **リポジトリ単位で鍵を指定する方式** が推奨されている

#### 学び

* 127は「その行のどれかのコマンドが存在しない」
* 見た目のコマンドが原因とは限らない
* 公式のベースイメージの構成はバージョンで変わる
* 非推奨機能（apt-key）を知るべき
