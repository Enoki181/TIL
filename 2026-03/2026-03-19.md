## Neonの疎通確認方法
Neonの公式ドキュメントでは、下記のように記述があり、手順が示されていた。
> Run the following command to create a controller and view. The controller will query the database version and supply it to the view file to render a web page that displays the PostgreSQL version. 

出典：Connect a Ruby on Rails application to Neon Postgres
　　　https://neon.com/docs/guides/ruby-on-rails

これに倣って、既存のTOPページのcontrollerとviewファイルに、下記記述を追記する。

【controller】
```
# === DEBUG: Neon Connection Check (Remove after verification) ===
    @postgres_version = ActiveRecord::Base.connection.execute("SELECT version();").first['version']
		@user_count = User.count
# ================================================================
```
【view】
```
<!-- === DEBUG: Neon Check === -->
<% if @postgres_version %>
  <p>DB Version: <%= @postgres_version %></p>
  <p>User Count: <%= @user_count %></p>
<% end %>
<!-- ========================= -->
```
これにより、
- Postgre Versionの表示 -> NeonPostgresとの疎通確認
- User.count = 0 の表示 -> テーブルからデータをとれることの確認

なお、デプロイ前にローカルで下記を実行しUserモデルを作成する。
```
rails g model User display_name:string line_user_id:string
```
そのうえで、マイグレーションファイルにnull: falseなど制約を入れた状態でdb:migrateをし、正常にテーブルが作成されることを確認する。
※ デプロイ前に、ローカルで`db:migrate:status`を実行・確認し、未実行のマイグレーションがない状態にする。

## Neon接続設定時のDATABASE_URLどこに書くか問題
当初.envにDATABASE_URL(NeonのConnectionString)を書こうかと思っていた。
しかし、開発環境で本番DBに接続されるケースがあると知った(Railsは環境変数に`ENV['DATABASE_URL']`があると優先して使用する)。
そのため、DATABASE_URLはRender環境変数で記載するのみとする。
なお、database.ymlにおいては、productionでDATABASE_URLを使用するよう記述を修正する。
