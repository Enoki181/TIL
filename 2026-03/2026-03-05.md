### LIFFミニアプリをRails + Docker + ngrokでやってみたらめちゃ苦戦した。Helloが遠い…

#### やりたかったこと

Railsで簡単なLIFFページを作り、LINEログインしてユーザー名を表示する。

```html
<h1 id="name">Loading...</h1>

<script>
async function main() {
  await liff.init({ liffId: "<%= ENV['LIFF_ID'] %>" });

  if (!liff.isLoggedIn()) {
    liff.login();
    return;
  }

  const profile = await liff.getProfile()

  document.getElementById("name").innerText =
    "Hello " + profile.displayName
}

main()
</script>
```

---
**【詰まったところ 1】`.env`の環境変数がDockerコンテナに渡っていない**

`.env`

```env
LIFF_ID=xxxxx
```

これを書いても、そのままでは **Dockerコンテナ内のRailsには渡らない。**

`compose.yml` に環境変数を追加する必要がある。

```yaml
web:
  environment:
    LIFF_ID: ${LIFF_ID:-}
```

その後コンテナを再起動。

```bash
docker compose restart
```

確認

```ruby
puts ENV["LIFF_ID"]
```

値が出ればOK  
※もしnilが出たら渡せてない

---
**【詰まったところ 2】ngrokのトンネルが落ちている**

無料ngrokは **ターミナルを閉じるとトンネルが終了**する。

エラー

```
ERR_NGROK_3200
The endpoint ... is offline
```

対処

```bash
ngrok http 3000
```

新しいURLが発行される。

例

```
https://xxxxx.ngrok-free.app
```

---
**【詰まったところ 3】RailsのHost Authorizationにブロックされる**

ngrok URLでアクセスすると

```
Blocked hosts: xxxxx.ngrok-free.app
```

対処

```ruby
# config/environments/development.rb

config.hosts << "xxxxx.ngrok-free.app"
```

コンテナ再起動。

---
**【詰まったところ 4】LIFF URLではなく普通のURLで開いている**

LIFFは

```
https://xxxxx.ngrok-free.app/liff
```

ではなく

**LINE Developersの「LIFF URL」** から開く必要がある。

例

```
https://liff.line.me/xxxxx
```

これでLINE内ブラウザで開く。  
※KeepメモにURLを貼り付けて開くとよい。

---
**【詰まったところ 5】エンドポイントURLのタイポ**

例

```
https://xxxxx.ngrok-free.app//liff
```

`//`になっていたので

* 正しくルーティングされない
* JSが動かない
* `Loading...` のままになる

正しいURL

```
https://xxxxx.ngrok-free.app/liff
```

### ドメイン駆動設計：トランザクションスクリプト
**トランザクションスクリプト**とは
- 単純な業務ロジックを実装するパターン
- DDDでいう**補完的業務領域**に向く

【特徴】
- 業務ロジックをトランザクション単位の処理として実装する
- ドメインモデルは薄い
- 処理単位でトランザクションを管理

【目的】
- データ不整合を防ぐ
- 処理途中の失敗でもDBを壊さない
