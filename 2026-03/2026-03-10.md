## database.yml とDB接続設定

ActiveRecordは `config/database.yml` に定義された設定を用いてDBに接続する。

```yaml
default: &default
  adapter: postgresql
  encoding: unicode

development:
  <<: *default
  database: app_development
```

* `&default`

  * エイリアス（別名）定義
* `*default`

  * 参照
* `<<: *default`

  * エイリアス内容を展開

この仕組みにより **環境ごとの設定重複を避ける（DRY）** ことができる。

### DBコンソール

```
rails dbconsole
```

* `database.yml` の設定を利用してDBクライアントを起動する
* PostgreSQLの場合は `psql` が起動

※VSCodeでは **SQLTools拡張**でもDB接続可能

## ActiveRecordとSQL

RailsではActiveRecordメソッドを利用してDBへ問い合わせる。

```ruby
Book.all
```

内部ではSQLが生成されている。

例

```sql
SELECT "books".* FROM "books"
```

Railsでは通常SQLを直接書かず、**ORM（ActiveRecord）を介してDB操作を行う。**

## Scaffolding

ScaffoldはCRUDアプリの基本構造を自動生成する機能。

例

```
rails g scaffold Book title:string author:string
```

生成される主な要素

* Model
* Controller
* View
* migration
* routes

Railsの**MVC構造をまとめて生成する開発補助機能**。

## resourcesルーティング

`resources` を使うとRESTfulルーティングをまとめて定義できる。

```ruby
resources :books
```

生成されるルート例

```
GET    /books
GET    /books/:id
POST   /books
PATCH  /books/:id
DELETE /books/:id
```
