### 作業#01 Neon接続用にデータ編集

1. `DATABASE_URL` を使う前提で `config/database.yml` を整理
- DB名などがあると、DATABASE_URLと競合する可能性があるので、database.ymlのproductionでは`default`を挿入せず、下記のように書き換えた。

```yaml
production:
  primary: &primary_production
    adapter: postgresql
    encoding: unicode
    url: <%= ENV['DATABASE_URL'] %>
  cache:
    <<: *primary_production
  queue:
    <<: *primary_production
  cable:
    <<: *primary_production
```

2. Neonの疎通確認の準備
- 既存のstatic_pagesのコントローラーとビューをデバッグ用に編集し、下記をTOPページに表示されるようにする。
  - PostgreSQLのバージョン　※Neonの公式ドキュメント準拠
  - User.countの値(今回CRUD機能を実装していないため、0と表示されるはず)

【コントローラーのTOPアクションへの追記】
```
# === DEBUG: Neon Connection Check (Remove after verification) ===
    @postgres_version = ActiveRecord::Base.connection.execute("SELECT version();").first['version']
    @user_count = User.count
# ================================================================
```

【topページのビューへの追記】
```
<!-- === DEBUG: Neon Check === -->
<% if @postgres_version %>
  <p>DB Version: <%= @postgres_version %></p>
  <p>User Count: <%= @user_count %></p>
<% end %>
<!-- ========================= -->
```
※コードはサンドイッチ形式で、変更範囲をわかりやすくした。
【備忘】次回作業時に、まずこれらの記述を削除する。

## Renderでデプロイ中にエラーが発生
database.ymlでproduction部の記述を下記のようにし手デプロイしたところエラーが発生し、デプロイが完了しなかった。
```
production:
    <<: *default
    url: <%= ENV['DATABASE_URL'] %>
```

【エラーメッセージ】
```
[89] ! Unable to load application: ActiveRecord::AdapterNotSpecified: The `cable` database is not configured for the `production` environment.

/opt/render/project/.gems/ruby/3.4.0/gems/activerecord-8.1.2/lib/active_record/database_configurations.rb:231:in 'ActiveRecord::DatabaseConfigurations#resolve_symbol_connection': The `cable` database is not configured for the `production` environment.
(ActiveRecord::AdapterNotSpecified)
```
すなわち、`cable`DBを探しているのにない！というエラー。

**原因**：
- database.ymlで下記記述を消して、3つのDBをなくしたこと
- そのうえで、`config/environments/production.rb`を変更しなかったこと
  - デフォルトで入っている3つのDBについて、本番環境で使用する記述がされたままだった。

```yaml
  cache:
    <<: *primary_production
    database: myapp_production_cache
    migrations_paths: db/cache_migrate
  queue:
    <<: *primary_production
    database: myapp_production_queue
    migrations_paths: db/queue_migrate
  cable:
    <<: *primary_production
    database: myapp_production_cable
    migrations_paths: db/cable_migrate
```
config/environments/production.rbを、3つのDB削除に合わせて修正するのではなく、デフォルトの記述に戻し、3つのDBを復活させることで対応。
修正後のdatabase.ymlの記述は上記の作業#01参照。
