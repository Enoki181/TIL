# Tech Log

## TailwindCSSのv4でのプラグインを書く場所・書き方
TailwindCSSのv4では、v3とプラグインの書き方が異なるので注意。
最初は、v3の記事を参考にしていたが、v4では設定ファイルの存在意義自体が変わっていることに気づいた。

- tailwind.config.js（JSによる設定）が必須ではなくなり、CSSファイル内で`@import "tailwindcss";` と`@plugin "daisyui";`を記述する形式になった。
- daisyUI v5の[公式ドキュメント](https://daisyui.com/docs/install/)では、TailwindCSSのv4の書き方に倣ってインストール方法を書いているので参考になる。公式ドキュメントの記述方法に倣い、今回は下記のようにapp/assets/stylesheets/application.tailwind.cssでテーマを追記する形にしている。

※実際に用いるテーマは、今後UIコンセプト設計時に自作する。
```
@plugin "daisyui" {
  themes: light --default, dark --prefersdark, lemonade;
}
```

エディタ（VSCode）が @plugin という新しいCSSアットルール（at-rule）を認識できず警告を出すことがあるが、Tailwindのビルド（CLI）が解釈できていればスルーでOK。
