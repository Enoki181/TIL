# Tech Log

## Rails render・locals・collection の整理

『Ruby on Railsアプリケーションプログラミング』で次のコードを見た。

```erb
<%= render book %>
```

`_book.html.erb` が呼ばれることは分かるが、フルで書いたときのコードについて、下記がわからなかった。
* `book: book` のようなパラメータはどこから来るのか
* `locals` とは何か
* `render @book` / `render @books` との違い
* 配列を`locals`で渡すのと何が違うのか

結論：
**「partial + locals」を省略した書き方になっている。**

---

### 1. `render` の基本形

フルの書き方

```erb
<%= render partial: "books/book", locals: { book: @book } %>
```

意味

| 要素            | 内容                              |
| ------------- | ------------------------------- |
| `partial:`    | 呼び出すテンプレート                      |
| `locals:`     | パーシャルに渡すローカル変数                  |
| `book: @book` | `_book.html.erb` 内で `book` が使える |

つまりパーシャルでは、下記のように書ける。

```erb
<%= book.title %>
```

イメージ
```
_book.html.erb 内部

book = @book
```

---

### 2. `book: book` の意味

`book: book`は Ruby の Hash。

```ruby
{ key: value }
```
つまり、下記のような役割をもつ。
※keyは任意の値だけど、`locals`のキーはパーシャルで使うローカル変数名になる。

```
book: book
│     │
│     └ 値（渡すオブジェクト）
└ キー（ローカル変数名）
```

Railsはこれを`locals: { book: book }`として解釈し、ローカル変数をパーシャルに渡している。

---

### 3. locals は省略できる

Railsでは下記のように、 `locals` を省略できる。

```erb
<%= render "books/book", book: @book %>
```

Rails内部では下記のように解釈される。

```
book: @book
↓
locals: { book: @book }
```

---

### 4. `render book`（Object Rendering）

Railsは **オブジェクトから partial名 と locals名 を推測できる**。

```erb
<%= render book %>
```

【Railsが自動で推測する内容】

| 推測       | ルール                   |
| -------- | --------------------- |
| partial  | モデル名 → `"books/book"` |
| localsキー | モデル名 → `book`         |
| locals値  | 渡されたオブジェクト            |

つまり内部では下記が実行されている。

```
render partial: "books/book",
       locals: { book: book }
```

---

### 5. locals に渡せる値

`locals`はRubyのHashなので、値は任意のオブジェクトを渡せる。

例
- 単一オブジェクト `book: @book`
- 配列 `books: @books`
- 複数値 `book: @book, user: current_user`
- ハッシュ `options: { editable: true }`

---
### 6. 配列の場合（Collection Rendering）

配列を渡すと Rails は collection として処理する。

```erb
<%= render @books %>
```

Rails内部では下記のように解釈される。

```
render partial: "books/book",
       collection: @books
```

そのため、実質的には、下記のような処理がおこなわれる。

```
@books.each do |book|
  render partial: "books/book", locals: { book: book }
end
```

---
### 7. `locals` で配列を渡す場合との違い

**locals**
```erb
<%= render "books/list", books: @books %>
```

**partial**
```
books.each do |book|
```
つまり、`books = @books`という1つの変数を渡すだけ。

---
**collection**
```erb
<%= render @books %>
```

つまり、Railsが「1レコード → 1パーシャルとして」くり返し描画する。

---
### 8. `render` の3パターン

| 書き方             | 渡すもの     | Railsの処理         |
| --------------- | -------- | ---------------- |
| `render book`   | 単一オブジェクト | partial + locals |
| `render @book`  | 単一オブジェクト | partial + locals |
| `render @books` | 配列       | collection       |

---
### 個人的理解メモ

```
@book   controller → view
book    partial内のローカル変数
locals  partialに変数を渡す仕組み
render  partialを描画する
```
