# Tech Log

## ローカル環境の実機確認のため、ngrokの固定ドメインを活用
LIFFアプリはhttpsじゃないと動かないので、ngrokを導入してトンネルを通すことで、ローカル環境でも実機で確認を行えるようにする。
ngrokでは、無料枠で1つの固定ドメイン(static domain)を利用可能なので、今回はこれを使う。

**【固定ドメインを使う理由】**
起動ごとにURLが変動するのにあわせてLIFFアプリのエンドポイントURLを更新するという手間を解消できる。

### RailsのHost保護回避
environments/development.rbに、下記を記述してngrokドメインを追加しないと、RailsのBlocked hostsエラーが発生した。
```
config.hosts << /.*\.ngrok-free\.dev/
config.hosts << /.*\.ngrok\.dev/
config.hosts << /.*\.ngrok-free\.app/
```

## エンドポイントURLの末尾スラッシュ
LIFF設定で末尾の / を忘れると、正しくroot（/）へルーティングされない。
